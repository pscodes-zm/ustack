# UStack Backend Development Prompt

## Project Overview

Build the backend for UStack, a Bitcoin savings platform designed for students and young savers. Users can create savings vaults, deposit Bitcoin through the Lightning Network, save toward goals, track progress, and optionally enable price protection features in future versions.

The system should prioritize simplicity, security, scalability, and fast onboarding.

---

# Core Technology Stack

## Backend

* Node.js (TypeScript preferred)
* Express.js or NestJS

## Database

* PostgreSQL

## Caching & Queues

* Redis

## Authentication

* OTP-based authentication
* JWT access tokens
* Refresh tokens

## Bitcoin Infrastructure

* Blink API for Lightning deposits and withdrawals

## Mobile Money

* Airtel Money API
* MTN MoMo API (future)
* Zamtel Money API (future)

## Deployment

* Replit (development)
* Railway / Render / VPS (production)

# User Management System

## Registration

Users can register using:

* Phone Number (preferred)
* Email (optional)
* Username

Store:

* user_id (UUID)
* username
* encrypted_phone
* encrypted_email
* account_status
* created_at
* updated_at

### Requirements

* No passwords
* OTP authentication only
* OTP expires after 5 minutes
* Maximum 5 OTP requests per hour

---

# Authentication Flow

## Sign Up

1. User enters username.
2. User enters phone number.
3. OTP sent.
4. User verifies OTP.
5. Account created.
6. JWT issued.

## Login

1. User enters phone number.
2. OTP sent.
3. User verifies OTP.
4. JWT issued.

# Vault Actions

## Create Vault

User provides:

* Vault name
* Vault type
* Goal amount

System creates vault.

---

## Deposit Into Vault

User selects:

* Vault
* Amount

System:

* Deducts from available balance
* Adds to vault balance
* Logs transaction

---

## Withdraw From Vault

System checks:

* Vault type
* Lock status
* Penalty requirements

If approved:

* Deduct vault balance
* Move funds to available balance

# Lightning Integration

## Blink Integration

Use Blink API for:

* Deposits
* Withdrawals
* Balance checks

### Deposit Flow

1. User selects Deposit.
2. Backend generates Lightning invoice.
3. Invoice displayed.
4. Blink webhook confirms payment.
5. User balance updated.

### Withdrawal Flow

1. User enters Lightning invoice.
2. Backend validates invoice.
3. Blink sends payment.
4. Transaction logged.
5. User balance updated.

---

# Price Protection System

Users may enable price protection.

## Settings

User selects:

* Trigger: 2%, 5%, or 10%
* Protection Amount:

  * 25%
  * 50%
  * 75%
  * 100%

Store:

* protection_enabled
* trigger_percentage
* protection_percentage
* asset_state

---

## Trigger Rules

### 2% Drop

* Detect 2% drop
* Wait 3 minutes
* If still below threshold:

  * Activate protection

Return to BTC:

* Wait 5 minutes after recovery

---

### 5% Drop

* Detect 5% drop
* Wait 10 minutes

Return:

* Wait 10 minutes

---

### 10% Drop

* Detect 10% drop
* Wait 20 minutes

Return:

* Wait 20 minutes
